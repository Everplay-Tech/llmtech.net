3. Your Server

AiMANAC requires a backend server. This is where your API keys are stored (encrypted via IHP capsules) and where AI requests are processed. You own this server. Three ways to run it:

Option A: Railway (Recommended for individuals)

Railway is a cloud platform. You push code, it runs. Postgres, Redis, and your backend all in one place. Reachable from anywhere — phone on WiFi, cellular, anywhere.

Cost: ~$5-10/month based on usage.

1

Sign up at railway.com

2

Create a new project. Add a PostgreSQL database and a Redis instance from the dashboard.

3

Add a new service from your GitHub repo (Everplay-Tech/AiMANAC), set the root directory to /backend

4

Set environment variables: ENCRYPTION_KEY, JWT_SECRET, JWT_REFRESH_SECRET, SECRETS_PROVIDER=environment, NODE_ENV=production. Reference the Postgres and Redis URLs from the database services.

5

Add a custom domain. Use an opaque subdomain (not backend.mysite.com) — something like hq-a3f9c2.yourdomain.com. Point a CNAME record at it from your DNS provider.

6

In AiMANAC Settings, enter: https://your-subdomain.yourdomain.com/api/v1

Security note: Use an opaque subdomain. Don't use predictable names like backend.aimanac.example.com. Your server URL is a credential — treat it like one.

Option B: Docker on your own machine

Run the backend on any computer — your Mac, a Raspberry Pi, a $5/month VPS. Docker Compose bundles everything.

git clone https://github.com/Everplay-Tech/AiMANAC.git
cd AiMANAC/backend
cp .env.production.example .env
# Fill in ENCRYPTION_KEY, JWT_SECRET, JWT_REFRESH_SECRET
docker compose up -d

This starts the backend, PostgreSQL, and Redis. To reach it from your phone outside your home network, use Tailscale (mesh VPN) or a Cloudflare Tunnel.

Option C: Local network only

Same Docker setup, but only accessible on your WiFi. Simplest option if you only use AiMANAC at home or work.

In AiMANAC Settings, enter: http://YOUR_SERVER_IP:3000/api/v1

Environment Variables

Three required secrets. Generate each with: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

ENCRYPTION_KEY=       # 64-char hex — encrypts API keys via IHP capsules
JWT_SECRET=           # auth token signing
JWT_REFRESH_SECRET=   # refresh token signing

Database and Redis URLs are auto-configured by Docker Compose and Railway.

6. The Production System

The production system is modeled on music DAWs. Instead of audio tracks, you have AI processing lanes.

Four Lanes

◆ Lane 1 — Generation (Cyan) — ideas originate
● Lane 2 — Processing (Violet) — ideas get shaped
▲ Lane 3 — Analysis (Pink) — ideas get checked
■ Lane 4 — Output (Green) — results go out

Each lane holds up to 4 modules. Modules can:

  • Chain within a lane (module 1 output feeds module 2 input)
  • Route to other lanes (direct or send/copy)
  • Read from Library artifacts (shared buses)
  • Post results to chat channels
  • Save to Library as artifacts

4 workflows per workspace. 4 lanes per workflow. 4 modules per lane. 64 modules max.

You build it, HELM runs it. Or HELM builds it and HELM runs it. Or you build it and you run it. Your call — think laterally.

Module Types

  • AI Prompt — Send text to a provider. Supports Extended Thinking, reasoning modes, vision.
  • Transform — AI-powered data reshaping via plain English instructions.
  • Conditional — AI evaluates output semantically and routes to different lanes based on the result.
  • Delay — Wait before the next step. 5 seconds to 1 hour.
  • Approval Gate — Pause for human confirmation before continuing.
  • Artifact Save — Save output to Library with tags.

7. HELM

HELM is the home screen. Two tabs.

Chat Tab

Talk to your AI. With a cloud provider connected, HELM has exclusive access to 6 production tools:

  • getProductionStatus — view all workflows, modules, and run history
  • triggerWorkflow / triggerModule — start a run
  • pauseWorkflow / cancelWorkflow — stop a run
  • routeToLane — inject content into a specific lane

Regular workspace chat channels have full AI capabilities but no production tools. HELM is the only interface with tool access.

Controls Tab

  • Provider Selector — switch between local and cloud AI
  • Autonomy Dial — 6 levels controlling tool execution permissions
  • Production Status — live workflow and module status
  • Cost Monitor — per-provider spending
  • MCP Server — desktop AI integration (Claude Code, Cursor)

Autonomy Levels

  • 0 OFF — Tools disabled. Text-only responses.
  • 1 SUGGEST — HELM proposes tool calls. You approve each one.
  • 2 ASK — Shows intent preview with cost estimate before executing.
  • 3 SMART (default) — Read-only tools auto-execute. Mutations require confirmation.
  • 4 NOTIFY — All tools auto-execute with notifications.
  • 5 FULL AUTO — Full autonomy. Circuit breaker catches runaway loops.

8. Security & Privacy

IHP Capsule Protocol: API keys are encrypted using cryptographic capsules with mathematical data flow enforcement. A key classified as LOCAL physically cannot decrypt when used for a CLOUD destination. This is cryptographic enforcement, not an application-level check.

SecurityBar

The bar at the bottom of HELM shows real-time data flow:

  • // local :: no egress — on-device AI, nothing left the phone
  • // cloud :: anthropic.com — data sent to a cloud provider

Tap to expand the Privacy Dashboard — LOCAL/LAN/CLOUD breakdown and recent connections.

Trust Profiles

Set allowed data flow classes per workspace. A workspace restricted to LOCAL+LAN will reject any cloud request. The capsule refuses to decrypt. The math says no.

What We Don't Do

  • No telemetry or analytics (Firebase/Sentry disabled by default)
  • No phone-home or license validation
  • No API key access from the mobile app (keys live on your server)
  • No user data collection or sharing

9. Troubleshooting

"Network error" in HELM

Phone can't reach the backend server.

  • Is the server running? (docker compose ps)
  • Are phone and server on the same WiFi?
  • Is the URL correct? Needs /api/v1 at the end.
  • Use the server's LAN IP, not localhost.

"Provider not initialized"

The provider needs an API key. Settings → AI Providers → Setup → paste a valid key.

Phi-4 Mini not available

Settings → On-Device AI → Download (~2.3 GB, WiFi required). After download, tap Activate. Fully offline after that.

Workflows won't run

Workflows require a cloud provider with tool access. Phi-4 handles conversation but cannot execute tools. Connect Claude or OpenAI in HELM Controls.

Database errors

cd backend
npx prisma migrate deploy

Fresh database:

npx prisma db seed

Starting over

Delete the app and reinstall. Onboarding runs again. Backend data is safe on your server — the app is the window, not the building.

Server reset: docker compose down && docker compose up -d
