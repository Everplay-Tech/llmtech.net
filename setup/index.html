1. What You Just Bought (Congratulations)

So here you are. You paid $9.99 — once, forever, like buying a book or a really good sandwich — and now you own an AI management system. Not a subscription. Not a "free tier" with a gun to your head. You own this thing.

You bought a synthesizer. You did not rent one. This distinction matters more than you think.

AiMANAC does not phone home. It does not track you. It does not sell your prompts to the highest bidder so they can train a model that replaces you. There is no telemetry. There is no DRM. There is no license server watching you sleep.

What you get:

  • 10 AI providers wired into one interface — swap between them like changing channels
  • On-device AI (Phi-4 Mini) — a neural network living in your phone, free as air, needs no internet
  • HELM — your AI command center with real production control tools
  • A 4-lane production system modeled on music DAWs — because "workflow builder" is a phrase that puts people to sleep
  • IHP cryptographic security — your keys are mathematically bound to their intended destinations. Not an if-statement. Math.
  • Two themes — Terminal (for the hackers) and Executive (for the people who sign the checks)

Everything runs on your hardware. Your phone. Your server. Your API keys on your machine. We are not in the loop. We don't want to be in the loop. The loop is yours.

3. Your Server, Your Rules

AiMANAC needs a backend server. "But wait," you say, "I thought there was no subscription!" There isn't. You run your own server. On your computer. On a Raspberry Pi. On a $5/month VPS. On that old laptop in your closet. Wherever you want.

This is where your API keys live (encrypted, not in plaintext — we're not animals). This is where AI requests get processed. This is the trust boundary. Everything inside it is yours.

The Docker Way (Recommended)

Four commands. That's it. I've seen people spend longer choosing a font.

git clone https://github.com/Everplay-Tech/AiMANAC.git
cd AiMANAC/backend
cp .env.example .env
# Edit .env — set ENCRYPTION_KEY (any random 64-char hex string)
docker compose up -d

This spins up three things in nice little containers:

  • AiMANAC Backend on port 3000 — the brain stem
  • PostgreSQL 14 with pgvector on port 5432 — where memories live
  • Redis 7 on port 6379 — the fast-twitch muscle fiber

Point Your Phone at It

1

Find your server's IP address (ifconfig or ip addr — look for something like 192.168.1.50)

2

In AiMANAC, go to Settings

3

Under Backend Server, type: http://192.168.1.50:3000/api/v1

4

Tap Connect — the dot goes green, and you're in business

Important neighborhood rule: Your phone and server need to be on the same WiFi network. For remote access, set up a VPN or reverse proxy. Don't just expose port 3000 to the internet unless you enjoy chaos.

The Minimum .env

Six lines between you and a working server. Fewer lines than a haiku, if haikus needed encryption keys.

NODE_ENV=production
PORT=3000
DATABASE_URL=postgresql://user:pass@localhost:5432/aimanac
REDIS_URL=redis://localhost:6379
ENCRYPTION_KEY=your-64-char-hex-string
JWT_SECRET=your-jwt-secret

5. Workspaces & Channels (Or: Where Things Live)

Workspaces are rooms. Each room has its own conversations, its own workflows, its own AI defaults. Think of them like projects, departments, moods — whatever organizational metaphor doesn't make you want to take a nap.

Each workspace gives you:

  • Chat Channels — separate conversations with AI. Like group chats, but useful.
  • Production View — the 4-lane workflow system (more on that next)
  • Settings — default provider, system prompt, budget cap

Creating one: Sidebar → Workspaces → the + button. Name it. Pick an icon. Done.

Each channel can override the workspace's default provider. Long-press the provider pill in chat to hot-swap mid-conversation. Because sometimes you need a different tool for a different job, and that's perfectly fine.

6. The Four-Lane Machine

Now we're getting into the weird and wonderful part. Most apps call this "workflow automation." We call it a production system, because it's modeled on music DAWs — Digital Audio Workstations. Ableton Live. AKAI MPC. Modular synthesizers with patch cables going everywhere.

Instead of audio signals, you're routing AI outputs. Same energy. Same joy.

The Four Lanes

Every workflow has exactly four lanes. Not three. Not five. Four. Like the elements, the seasons, the Beatles at their best:

◆ Lane 1 — Generation (Cyan) — where ideas are born
● Lane 2 — Processing (Violet) — where they get shaped
▲ Lane 3 — Analysis (Pink) — where they get checked
■ Lane 4 — Output (Green) — where they go out the door

Each lane holds up to 4 modules. Modules can:

  • Chain within a lane (module 1 feeds module 2 feeds module 3 — like pedals on a pedalboard)
  • Route to other lanes (direct send or copy, like aux sends on a mixing board)
  • Read from Library artifacts (shared buses — communal knowledge)
  • Post results to chat channels (your AI talking to your other AI's room)
  • Save to Library as artifacts (reusable outputs)

The Module Types

Six flavors. Each one does exactly one thing well:

  • AI Prompt — Send text to any provider. Claude Extended Thinking, OpenAI reasoning, vision. The workhorse.
  • Transform — Tell it what you want in plain English: "summarize this," "convert to JSON," "make it sound like a pirate." AI-powered reshaping.
  • Conditional — AI reads the output and makes a judgment call: does this meet the criteria? Route accordingly. Semantic evaluation, not regex.
  • Delay — Wait. Sometimes that's the right move. 5 seconds to 1 hour.
  • Approval Gate — Stop and wait for a human to say "yes." Because sometimes a machine shouldn't decide alone.
  • Artifact Save — Save the output to Library with tags. Build up a knowledge base, one run at a time.
4 workflows per workspace. 4 lanes per workflow. 4 modules per lane. That's 64 possible modules humming along. Not infinite. Deliberately finite. Constraints breed creativity.

7. HELM — The Captain's Chair

HELM is the home screen. It's where you sit when you want to talk to your AI and actually do things. Two tabs. No submenus. No hidden panels. Two tabs.

Chat Tab

Talk to your AI. Simple as that. But here's the trick: when you have a cloud provider connected, HELM gets exclusive access to production tools. Six of them:

  • getProductionStatus — "Show me everything. All workflows. All modules. What's running, what's broken."
  • triggerWorkflow / triggerModule — "Run this thing."
  • pauseWorkflow / cancelWorkflow — "Stop. Wait. No, really stop."
  • routeToLane — "Take this text and inject it into the Cyan lane."

Regular workspace chat channels get full AI capabilities — reasoning, analysis, code generation — but they do not get these tools. HELM is special. HELM is the bridge. Regular channels are the crew quarters. Both important. Different access.

Controls Tab

The dashboard. The cockpit. The mixer board:

  • Provider Selector — switch between local and cloud AI with one tap
  • Autonomy Dial — how much freedom does HELM get? (see below)
  • Production Status — live view of all workflows and modules
  • Cost Monitor — what you're spending, per provider, in real time
  • MCP Server — connect desktop AI tools (Claude Code, Cursor) to your system

The Autonomy Dial (This Is Important)

The dial has six positions. Like a volume knob, but for trust:

  • 0 — OFF — Tools disabled. HELM can talk but not touch. Training wheels.
  • 1 — SUGGEST — HELM proposes tool calls. You approve each one. Very polite.
  • 2 — ASK — Shows you what it wants to do with a cost estimate. Informed consent.
  • 3 — SMART (default) — Read-only tools auto-execute. Anything that changes things asks first. The sensible middle ground.
  • 4 — NOTIFY — Everything auto-executes, but you get notifications. "Hey, I just did this."
  • 5 — FULL AUTO — Full autonomy. The circuit breaker is still active (it catches runaway loops), but HELM drives. Buckle up.
Start at 3. That's what it's there for. Move the dial when you're ready. Nobody's judging.

8. The Vault (Security & Privacy)

We need to talk about security. Not because it's scary, but because we did something unusual and you deserve to know about it.

IHP Capsule Protocol: Your API keys are not stored in plaintext. They're encrypted using cryptographic capsules with mathematical data flow enforcement. A key classified as LOCAL physically cannot decrypt when used for a CLOUD destination. This isn't an if-statement checking a string. It's cryptography. The math won't let it happen. Like trying to put a square block through a round hole, except the hole is made of prime numbers.

The SecurityBar

That little bar at the bottom of HELM? That's your data flow monitor. In real time, it tells you exactly where your data is going:

  • // local :: no egress — everything stayed on your phone. Nothing left. Peace.
  • // cloud :: anthropic.com — data went to Anthropic's servers. You chose this. You know.

Tap it to expand the Privacy Dashboard — shows LOCAL/LAN/CLOUD breakdown and recent connections. No surprises. Ever.

Trust Profiles

You can set rules per workspace. "This workspace is LOCAL+LAN only." Try to call a cloud provider from that workspace? Denied. Not by an if-statement — by the capsule refusing to decrypt. The math says no.

What We Will Never, Ever Do

And I mean never. Not "never unless our investors make us." Never.

  • No telemetry, no analytics. Firebase and Sentry are in the code but disabled by default.
  • No phone-home. No license server. No heartbeat check.
  • No access to your API keys from the mobile app. Keys live on your server. Period.
  • No data collection. No usage tracking. No "anonymized" anything.
We built this for people who are tired of being the product. You paid ten dollars. That makes you the customer. Remember when that's how things worked? We do.

9. When Things Go Sideways

Things go sideways. It's the nature of software, and of life. Here's how to unsideway them:

"Network error" in HELM

Your phone can't find your server. Don't panic. Check these things, in order:

  • Is the server actually running? (docker compose ps — look for "Up")
  • Are phone and server on the same WiFi network? (This is the #1 cause. Every time.)
  • Is the URL in Settings correct? It needs /api/v1 at the end.
  • Try the server's LAN IP address, not localhost. Your phone doesn't know what your server calls "localhost."

"Provider not initialized"

The AI provider needs an API key. You haven't given it one yet, or the key expired, or it was for the wrong thing. Go to Settings → AI Providers → Setup → paste a valid key.

Phi-4 Mini says "not downloaded"

It needs to be downloaded first. Settings → On-Device AI → Download. ~2.3 GB. Needs WiFi for the download (after that, it's fully offline forever). Go make a sandwich. A good one.

Workflows won't run

Workflows need a cloud provider with tool access. Local AI (Phi-4) is brilliant but it can't pull triggers — it doesn't have hands, so to speak. Connect Claude or OpenAI in HELM Controls.

Database acting up

Run the migrations. This fixes most things:

cd backend
npx prisma migrate deploy

Fresh database? Also seed it:

npx prisma db seed

Still broken? Check your DATABASE_URL in .env. Then check it again. Then check if Postgres is actually running.

Everything is on fire

Take a breath. Delete the app. Reinstall it. The onboarding will walk you through setup again. Your backend data is safe on your server — the app is just the window into it.

If the server is the problem: docker compose down && docker compose up -d. Clean restart. Fresh containers. New day.
