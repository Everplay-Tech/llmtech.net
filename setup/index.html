1. What is AiMANAC?

AiMANAC is a one-time purchase ($9.99) AI management app. Unlike subscription services, you own it forever and run your own backend server. Your API keys, data, and conversations stay on your infrastructure.

Key principle: AiMANAC never phones home. Zero telemetry, zero DRM, zero license validation. You buy a synthesizer, not rent one.

What you get:

  • 10 AI providers managed from one interface
  • On-device AI (Phi-4 Mini) — works offline, free forever
  • HELM — AI command center with production control tools
  • 4-lane production system — DAW-style workflow automation
  • IHP cryptographic security — mathematical data flow enforcement
  • Dual themes — Terminal (hacker) and Executive (professional)

3. Setting Up Your Backend Server

AiMANAC requires a self-hosted backend. This is where your API keys are stored (encrypted via IHP capsules) and where AI requests are processed.

Quick Start with Docker

git clone https://github.com/Everplay-Tech/AiMANAC.git
cd AiMANAC/backend
cp .env.example .env
# Edit .env — set ENCRYPTION_KEY (any random 64-char hex string)
docker compose up -d

This starts:

  • AiMANAC Backend on port 3000
  • PostgreSQL 14 with pgvector on port 5432
  • Redis 7 on port 6379

Connect Your Phone

1

Find your server's IP address (e.g., 192.168.1.50)

2

In AiMANAC, go to Settings

3

In Backend Server, enter: http://192.168.1.50:3000/api/v1

4

Tap Connect — the status should turn green

Important: Your phone and server must be on the same network (WiFi). For remote access, set up a VPN or reverse proxy with HTTPS.

Minimum .env Configuration

NODE_ENV=production
PORT=3000
DATABASE_URL=postgresql://user:pass@localhost:5432/aimanac
REDIS_URL=redis://localhost:6379
ENCRYPTION_KEY=your-64-char-hex-string
JWT_SECRET=your-jwt-secret

5. Workspaces & Channels

Workspaces organize your AI conversations and workflows. Think of them as projects or departments.

Each workspace has:

  • Chat Channels — conversations with AI providers
  • Production View — 4-lane workflow system
  • Settings — default provider, system prompt, budget

Create a workspace: Sidebar → Workspaces → + button

Each channel can override the workspace's default provider. Long-press the provider pill in chat to change per-channel.

6. The Production System

AiMANAC's production system is modeled on music DAWs (Digital Audio Workstations). Instead of audio tracks, you have AI processing lanes.

The 4-Lane Arrangement

◆ Lane 1 — Generation (Cyan)
● Lane 2 — Processing (Violet)
▲ Lane 3 — Analysis (Pink)
■ Lane 4 — Output (Green)

Each lane holds up to 4 modules. Modules can:

  • Chain within a lane (module 1 output → module 2 input)
  • Route to other lanes (direct or send/copy)
  • Read from Library artifacts (shared buses)
  • Post results to chat channels
  • Save to Library as artifacts

Module Types

  • AI Prompt — Send text to an AI provider. Supports Claude Extended Thinking, OpenAI reasoning, vision.
  • Transform — AI-powered data reshaping ("summarize this", "convert to JSON")
  • Conditional — Route to different lanes based on semantic AI evaluation
  • Delay — Wait before next step (5s to 1hr)
  • Approval Gate — Pause and wait for human confirmation
  • Artifact Save — Save output to Library with tags

7. HELM — Your AI Command Center

HELM is the home screen. Two tabs:

Chat Tab

Talk to your AI. With a cloud provider connected, HELM has exclusive access to production control tools:

  • getProductionStatus — View all workflows and modules
  • triggerWorkflow / triggerModule — Start runs
  • pauseWorkflow / cancelWorkflow — Control execution
  • routeToLane — Inject content into a lane

Regular workspace chat channels retain full AI capabilities but do not have these tools.

Controls Tab

  • Provider Selector — Switch between local/cloud AI
  • Autonomy Dial — 6 levels controlling tool execution permissions (OFF → FULL AUTO)
  • Production Status — Live workflow/module status
  • Cost Monitor — Spending across providers
  • MCP Server — Desktop AI integration (Claude Code, Cursor)

Autonomy Levels

  • 0 OFF — Tools disabled. Text-only responses.
  • 1 SUGGEST — HELM suggests tool calls, waits for approval.
  • 2 ASK — Shows intent preview with cost estimate.
  • 3 SMART (default) — Read-only tools auto-execute. Mutations need confirmation.
  • 4 NOTIFY — All tools auto-execute with notifications.
  • 5 FULL AUTO — Full autonomy. Circuit breaker still active.

8. Security & Privacy

IHP Capsule Protocol: Your API keys are not stored in plaintext. They are encrypted using IHP capsules with cryptographic data flow enforcement. A key classified as LOCAL physically cannot decrypt when used for a CLOUD destination. This is mathematical enforcement, not an if-statement.

SecurityBar

The bar at the bottom of HELM shows real-time data flow:

  • // local :: no egress — Using on-device AI, nothing left the phone
  • // cloud :: anthropic.com — Data sent to a cloud provider

Tap it to expand the Privacy Dashboard showing LOCAL/LAN/CLOUD breakdown and recent connections.

Trust Profiles

Control which data flow classes are allowed per workspace. A workspace with only LOCAL+LAN allowed will reject any request that would touch a cloud provider.

What We Never Do

  • No telemetry or analytics (Firebase/Sentry disabled by default)
  • No phone-home or license validation
  • No API key access from the mobile app (keys live on your server)
  • No user data collection or sharing

9. Troubleshooting

"Network error" in HELM

Your phone can't reach the backend server. Check:

  • Is the server running? (docker compose ps)
  • Are phone and server on the same network?
  • Is the URL correct in Settings? (include /api/v1)
  • Try using the server's LAN IP, not localhost

"Provider not initialized"

The AI provider needs an API key. Go to Settings → tap Setup on the provider card → enter your key.

Phi-4 Mini says "not downloaded"

Go to Settings → On-Device AI → Download. Requires ~2.3 GB free space and a WiFi connection for the initial download.

Workflows won't run

Workflows require a cloud provider with tool access. Local AI (Phi-4) cannot execute tools. Connect Claude or OpenAI, then select it in HELM Controls.

Backend database errors

Run migrations:

cd backend
npx prisma migrate deploy

If using a fresh database, also seed:

npx prisma db seed
